L1p-value vs α — the two numbers everybody mixes up

🧠 Simple explanation (very easy English)

You make a guess: "Nothing changed." This is called H₀ (H-zero, the null guess).

Then you look at your data. The p-value is a number between 0 and 1. It answers ONE question:

"If H₀ were really true, how often would I see data this surprising?"

Small p-value (close to 0) → my data is super surprising → maybe H₀ is wrong.

Big p-value (close to 1) → my data is normal → H₀ is fine.

α (alpha) is the line in the sand. You pick it BEFORE looking at data. Usually α = 0.05.

Rule: if p < α → reject H₀. If p ≥ α → fail to reject H₀.

📐 Math language (exam style — use these exact words)
  • Null hypothesis (H₀): the default claim about a parameter (often "no effect" or "the value equals μ₀").
  • Alternative hypothesis (H_a): the claim you would believe if H₀ is rejected.
  • Significance level (α): the maximum Type I error rate you will tolerate; chosen before the test.
  • p-value: P(observing data at least as extreme as ours, given H₀ is true).
  • Decision rule: reject H₀ if p < α; otherwise fail to reject H₀.
p-value = P( T  more extreme than  t_observed  |  H₀ true )
🔗 Connection (idea ↔ math)
Plain EnglishExam phrasing
"How weird is my data if nothing changed?"p-value
"How weird is too weird?"α (significance level)
"My data is too weird — H₀ probably wrong"Reject H₀
"Not weird enough — can't accuse H₀"Fail to reject H₀

Key: p-value comes from the DATA. α comes from YOU. They are not the same thing — they only meet at the comparison.

🍎 Intuitive example (step by step)

Your friend says: "I always flip a fair coin." You don't trust them. You flip it 10 times and it lands heads 9 times.

  1. H₀: coin is fair (P(heads) = 0.5). H_a: coin is unfair.
  2. Pick α = 0.05 (a 5% false-alarm tolerance).
  3. Compute p-value: under a fair coin, P(at least 9 heads in 10 flips) ≈ 0.011.
  4. Compare: 0.011 < 0.05 → reject H₀. The coin is probably not fair.

Read out loud: "If the coin really were fair, we'd see 9-or-more heads only 1.1% of the time. That's rare enough to doubt fairness."

L2Type I, Type II, β, and Power — the four error words

🧠 Simple explanation (very easy English)

A jury can be wrong in two ways:

  • Convict an innocent person → BAD ERROR #1 = Type I
  • Let a guilty person go free → BAD ERROR #2 = Type II

Translate to stats:

  • Type I: reject H₀ when H₀ is actually TRUE. (Said "effect" when there was none.)
  • Type II: fail to reject H₀ when H₀ is actually FALSE. (Missed a real effect.)

Power = how good your test is at catching real effects. Higher power = fewer Type II errors.

📐 Math language (exam style)
  • α = P(reject H₀ | H₀ true) = Type I error rate.
  • β = P(fail to reject H₀ | H₀ false) = Type II error rate.
  • Power = 1 − β = P(reject H₀ | H₀ false).
Power = 1 − β = P( reject H₀ | H_a true )

Four levers move power: bigger sample size n, bigger true effect, smaller noise σ, bigger α.

🔗 Connection (idea ↔ math)
Plain EnglishGreek letter / phrase
"False alarm rate"α (Type I)
"Miss rate"β (Type II)
"Catch rate"Power = 1 − β
"My test gets stronger when I…"increase n, increase effect, decrease σ, increase α
🍎 Intuitive example

You build a test that flags spam emails. You set α = 0.05. Out of 100 normal emails, your test wrongly flags 5 of them as spam. That is your Type I error rate (α).

Out of 100 actual spam emails, your test misses 20 of them (lets them through). That is β = 0.20. Your power = 1 − 0.20 = 0.80. You catch 80% of real spam.

If you tighten α to 0.01 (only flag really obvious spam), you get fewer Type I errors — but you also catch less real spam. Power drops. Classic trade-off.

✅ Recap (the 4 lines to memorize)

α = P(reject H₀ | H₀ TRUE) — the false alarm rate, you pick it (0.05).
β = P(fail to reject H₀ | H₀ FALSE) — the miss rate.
Power = 1 − β — the catch rate.
Trade-off: ↓α → ↓Power. ↑n → ↑Power. ↑σ → ↓Power. ↑effect → ↑Power.

⚠️ Common mistakes
Mistake 1 — Confusing Type I with Type II.
Mnemonic: "I" looks like 1 → α (the FIRST one you choose). "II" looks like 2 → β (the SECOND one, which falls out of the math).
Mistake 2 — Thinking lower α always = better test.
Lower α reduces false alarms but lowers power. There is always a trade-off.
Mistake 3 — Power = 1 − α.
No! Power = 1 − β. α and β are independent and live on opposite "truth columns."

L3Critical value t* — what it is, how to read a t-table, how to compute in R

🧠 Simple explanation

Imagine the t-distribution as a hill. Most of the area is in the middle. The two skinny tails are far from the center.

The critical value t* is the EXACT t-number that marks the edge of those tails.

If your test's t-statistic lands beyond t* → reject H₀. If it lands inside → fail to reject H₀.

t* depends on TWO things: (1) how strict you are (your α), (2) your degrees of freedom (df = n − 1).

📐 Math language
  • Critical value t*: the t-value such that P(|T| > t*) = α for a two-sided test.
  • Degrees of freedom (df): for a one-sample t-test, df = n − 1.
  • Confidence level C: 1 − α. So 95% CI corresponds to α = 0.05.
For two-sided CI at level C: t* = qt( (1 + C) / 2, df )

For α = 0.05 (95% CI), df = 14: t* = qt(0.975, 14) = 2.145.

🔗 Connection (idea ↔ math)
Plain EnglishR code
"the upper threshold for a 95% CI, df=14"qt(0.975, 14)
"the lower threshold (mirror)"qt(0.025, 14) = −t*
"the upper threshold for a 99% CI, df=9"qt(0.995, 9) = 3.250
🍎 Step-by-step example (Khan Academy classic)

"Find t* for a 98% CI from n = 15 observations."

  1. Confidence = 98% → α = 0.02 → α/2 = 0.01 in each tail.
  2. df = n − 1 = 15 − 1 = 14.
  3. Look up row df = 14, column "upper-tail 0.01" in a t-table → t* = 2.624.
  4. In R: qt(0.99, 14) = 2.624. Same answer.

✅ Recap

For a two-sided CI at confidence C, df = n − 1: t* = qt((1 + C) / 2, df).
Smaller df → bigger t* (fatter-tail correction).

⚠️ Common mistakes
Mistake 1 — Using α instead of (1 − α/2) in qt().
qt(0.05, 14) gives the LOWER cutoff (negative t*). The UPPER cutoff is qt(0.975, 14).
Mistake 2 — Forgetting df = n − 1.
df depends on the sample size minus one for a one-sample t. Two-sample formulas differ — be careful.
Mistake 3 — Using z (1.96) instead of t (2.064 etc.) for small n.
Z is fine when σ is known or n is huge. For real samples with unknown σ, use t.

L4Confidence Interval — the "range" view of the same decision

🧠 Simple explanation

You measured x̄ = 23.5 minutes. But the true mean might be a little different. A confidence interval (CI) is a range that probably contains the true mean.

"95% CI" = built so that, if you repeated the experiment 100 times, about 95 of the intervals you'd build would catch the true mean.

Big magic: rejecting H₀ at α = 0.05 happens EXACTLY when the claimed value μ₀ is OUTSIDE the 95% CI.

📐 Math language
CI = x̄ ± t* · ( s / √n )
  • x̄ = sample mean.
  • t* = critical value from Lesson 3 (depends on df and confidence).
  • s / √n = standard error of the mean.
  • t* · (s / √n) = margin of error.

Standard error shrinks as n grows: SE = s / √n.

🔗 Connection (idea ↔ math)
Plain EnglishExam phrasing
"how wide is the band of plausible means"margin of error = t* · SE
"how confident am I"confidence level C (1 − α)
"the claimed value is INSIDE the band"fail to reject H₀
"the claimed value is OUTSIDE the band"reject H₀
🍎 Step-by-step example

50 wait times: x̄ = 23.5, s = 8.0, n = 50, claim μ₀ = 20. Build the 95% CI.

  1. SE = s / √n = 8 / √50 ≈ 1.131.
  2. df = 49 → t* = qt(0.975, 49) ≈ 2.010.
  3. Margin = 2.010 × 1.131 ≈ 2.273.
  4. CI = 23.5 ± 2.273 → [21.23, 25.77].
  5. μ₀ = 20 is OUTSIDE [21.23, 25.77] → REJECT H₀ at α = 0.05.

✅ Recap

CI = x̄ ± t* · (s/√n). μ₀ inside CI ↔ fail to reject. Higher confidence → wider CI. Bigger n → narrower CI.

⚠️ Common mistakes
Mistake 1 — "95% probability the true mean is in MY interval."
Once you compute it, the interval either contains μ or doesn't. The 95% refers to the long-run procedure.
Mistake 2 — "CI is just for visualization, not for tests."
CI and a two-sided test are mathematically equivalent. If the null value is outside the CI, you reject.
Mistake 3 — Using s² (variance) instead of s (SD) in margin formula.
SE uses standard deviation, not variance. Don't square it.
