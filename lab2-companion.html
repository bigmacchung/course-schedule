What Is Hypothesis Testing?
+30 XP

Airport claims average TSA wait = 20 min. Travelers say it's longer. How do you settle this?

Two Competing Claims

Null Hypothesis H0: The status quo. "Mean wait time IS 20 minutes."

Alternative Hypothesis HA: "Mean wait time is NOT 20 minutes."

This is a two-sided test (could be higher OR lower).

The p-value

P(test statistic as extreme as observed | H0 true)

It is NOT:

x P(H0 is true)

x P(HA is true)

x The probability of making a Type I Error (that's α)

Decision Rule with α

Pick α (usually 0.05) BEFORE looking at data.

If p-value < α then Reject H0

If p-value ≥ α then Fail to reject H0

"Fail to reject" ≠ "H0 is true." Just means not enough evidence.

Colleague says p-value = 0.70 means '70% chance the true mean is 20.' Correct?
Yes, the p-value is P(H0 true)
No -- p-value is P(data this extreme | H0 true)
No -- p-value is P(HA true)
No -- p-value gives P(Type I Error)

The Sign Test
+35 XP

What if data is heavily skewed? The sign test tests the median -- no normality needed.

Sign Test vs t-Test

t-test: Tests the mean μ. Requires approx normal data.

Sign test: Tests the median μ̃. Works for ANY distribution.

Algorithm

1. Subtract μ0 from each data point

2. Count + signs and - signs (ignore zeros)

3. Under H0, each sign is a fair coin flip (p = 0.5)

4. Use Binomial distribution for p-value

Practice Quiz Q4b

Wait times: 13.2, 25.0, 8.7, 31.4, 17.9, 22.3, 9.5. Testing μ̃ = 20.

Signs: -, +, -, +, -, +, -

n+ = 3, n- = 4, n = 7

R Code: sign_test function (Practice Quiz Q5)
sign_test <- function(x, mu0 = 20){{
    signs <- x - mu0
    n_pos <- sum(signs > 0)
    n_neg <- sum(signs < 0)
    n <- n_pos + n_neg
    min_count <- min(n_pos, n_neg)
    p_val <- pbinom(min_count, size = n, prob = 0.5) * 2
    return(p_val)
}}
pbinom(k, size=n, prob=0.5) = P(X ≤ k) for Binomial(n, 0.5). Multiply by 2 for two-sided.
R Code: Q4c: Computing the p-value
"cm"># n+ = 3, n- = 4, min(3,4) = 3, n = 7
2 * pbinom(3, size = 7, prob = 0.5)
"cm"># = 1.0 (fail to reject)
Sign test quirk: discrete p-values

With n=7, the closest achievable α to 0.05 is 0.015625. The Binomial is discrete -- can't hit every α level.

Cohen's d -- Standardized Effect Size
+30 XP
Cohen's d (one-sample)

d = |x̄ - μ0| / s

d < 0.2 = small, 0.2 ≤ d < 0.8 = medium, d ≥ 0.8 = large

R Code: Computing in R (Lab 3 Q1.6)
cohens_d_func <- function(my_data, mu_0){{
    s <- sd(my_data)
    if(s == 0) return(NA)
    d <- abs(mean(my_data) - mu_0) / s
    if(d < 0.2) cls <- "small"
    else if(d < 0.8) cls <- "medium"
    else cls <- "large"
    return(c(d, cls))
}}

sign_test <- function(x, mu0){{
    signs <- x - mu0
    n_pos <- sum(signs > 0)
    n_neg <- sum(signs < 0)
    n <- n_pos + n_neg
    min_count <- min(n_pos, n_neg)

Key definitions

p-value = P(data this extreme | H0 true). NOT P(H0 true).

Type I Error = Reject H0 when true. Rate = α.

Type II Error = Fail to reject H0 when false. Rate = β.

Power = 1 - β = P(reject | H0 false). Depends on n, α, σ, δ.

Big Data Trap = Large n makes tiny effects significant. Check CI and effect size.

Cohen's d = |x̄ - μ0| / s. Small < 0.2, Medium 0.2-0.8, Large > 0.8.

Bootstrap CI: resample with replacement, compute mean, repeat, quantiles.

Sign test: Tests median. Uses Binomial(n, 0.5). Any distribution.

CI for testing: μ0 inside CI = fail to reject. Outside = reject.

t-test conditions: (1) Normality (esp. small n), (2) Independence.
